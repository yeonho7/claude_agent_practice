# 🎉 Git 실습 저장소

본 저장소에 방문해주셔서 감사합니다!

이곳은 Git과 GitHub의 기본 개념부터 고급 기술까지 학습할 수 있는 실습 공간입니다. 초보자부터 경험자까지 모두가 유용한 정보를 얻을 수 있도록 구성되어 있습니다.

## Git 학습을 위한 재미있는 이야기 😂

**Q: 왜 개발자는 화장실에 가는 걸 좋아할까요?**  
A: Git commit을 하기 전에 한 번 더 생각하고 싶으니까요! 🚽💡

**Q: 코드가 작동하지 않으면?**  
A: commit 메시지를 확인해보세요. 아마도 "빠른 수정" 또는 "이건 뭐하는 거야?"라고 적혀있을 가능성이 높습니다! 🤦

**Q: Git 히스토리를 지우고 싶을 때 어떻게 할까요?**  
A: 하지 말아야 할 것을 배우는 가장 좋은 방법입니다! 그것이 바로 `git reset --hard origin/master`를 친구에게 추천하지 말아야 하는 이유입니다! 😱

**Q: 병합 충돌(merge conflict)이 무섭나요?**  
A: 처음에는 무섭지만, 팀과 협업하는 개발자라면 결국 그것이 삶의 일부가 됩니다. 충돌을 해결하다 보면 Git의 진정한 가치를 깨닫게 될 것입니다! 💪

## Git 학습 가이드 📚

### Commit 메시지 작성
Git commit을 진행하기 전에, 충분한 시간을 갖고 변경 사항을 검토해주시기를 부탁드립니다. 좋은 commit 메시지는 다음 개발자(혹은 미래의 자신)가 코드 변경을 이해하는 데 큰 도움이 됩니다. 명확하고 간결하면서도 변경의 이유를 포함하는 메시지를 작성하려고 노력해야 합니다.

### 코드 검토 권장사항
코드가 예상대로 작동하지 않을 때는 commit 메시지를 참고하여 변경 사항을 확인하시기를 권장드립니다. 또한 관련된 pull request 설명과 댓글도 함께 확인하면 더 깊이 있는 이해가 가능합니다.

### 브랜치 전략
브랜치는 기능 개발, 버그 수정, 실험 등 다양한 목적으로 사용됩니다. 좋은 브랜치 전략을 수립하면 팀의 협업이 훨씬 수월해집니다. `feature/`, `bugfix/`, `hotfix/` 같은 접두사를 사용하여 브랜치의 목적을 명확히 하는 것이 좋습니다.

### 충돌 해결 방법
병합 충돌은 같은 파일의 같은 부분을 여러 브랜치에서 수정했을 때 발생합니다. 충돌을 두려워하지 마세요. Git은 충돌 부분을 명확히 표시해주므로, 어느 코드를 유지할지 신중하게 선택하고 테스트를 진행하면 됩니다.

## 실용적인 팁 🔥

- `git push --force`는 신중하게 사용하세요! 특히 공유 브랜치에서는 더욱 주의해야 합니다.
- `git revert`는 변경을 취소할 때 유용합니다. `git reset`과 다르게 히스토리를 유지하므로 협업할 때 안전합니다.
- merge conflict는 Git 학습의 기회입니다! 🎮
- `git stash`를 사용하면 작업 중인 변경사항을 임시로 저장했다가 나중에 복구할 수 있습니다.
- 좋은 커밋 습관을 들이면, 나중에 버그를 찾을 때 `git bisect` 명령어를 효과적으로 사용할 수 있습니다.
- 정기적으로 `git log --oneline`으로 히스토리를 확인하는 습관은 프로젝트 이해도를 높입니다.

## Git 기본 명령어 🛠️

자주 사용하는 Git 명령어들입니다:

- `git status` - 현재 작업 상태 확인
- `git add <file>` - 파일을 스테이징 영역에 추가
- `git commit -m "메시지"` - 변경사항 커밋
- `git push` - 로컬 브랜치를 원격 저장소에 업로드
- `git pull` - 원격 저장소의 변경사항을 가져오기
- `git branch` - 브랜치 목록 확인 및 생성
- `git checkout <branch>` - 다른 브랜치로 전환
- `git merge <branch>` - 다른 브랜치의 내용을 현재 브랜치에 병합
- `git log` - 커밋 히스토리 확인
- `git diff` - 변경 내용 비교

이러한 명령어들을 능숙하게 다루면 Git을 더욱 효과적으로 활용할 수 있습니다.
